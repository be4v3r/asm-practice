# Практична 12. Робота з пам’яттю: реверс масиву + копіювання блоків

**Мова:** Assembler i386 (NASM)  
**ОС:** Debian Linux  
**Ввід/вивід:** лише `int 0x80` (sys_read/sys_write/sys_exit)

## Мета
Копіювати блоки пам’яті, будувати реверс та перевіряти паліндром.

## Вхідні дані
n (5..200), далі n чисел.

## Вихідні дані
1) Вихідний масив
2) Реверсований масив
3) PALINDROME: YES/NO

## Що потрібно використати
- додатковий буфер для копії
- копіювання циклом (bonus: rep movsd)
- порівняння пар елементів
- адресація [base+idx*4]

## Персоналізація (обов’язково)
Додатково: перевірити паліндром після XOR кожного елемента з K (вивести другий результат).

### Позначення персональних параметрів
- `Y3` — третя цифра року народження (наприклад 2004 → 0)
- `Y4` — четверта цифра року народження (2004 → 4)
- `D` — день народження (1..31)
- `M` — місяць народження (1..12)
- `N` — порядковий номер літери **'n'** у прізвищі (1..len); якщо 'n' немає — `N = 3`
- `K = 1 + (Y3 + Y4) mod 9`
- `S = D + 10*M + 100*Y4`

## Критерії до коду
- Код компілюється NASM та лінкується `ld` (команди нижче).
- У коді є коментарі базових блоків: `I/O`, `parse`, `math`, `logic`, `loops`, `memory`.
- Немає коментарів, що згадують ChatGPT.
- Результат відповідає вимогам задачі та коректно обробляє описані граничні випадки.
